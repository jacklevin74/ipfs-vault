🔐 X1 Vault

Encrypted IPFS storage using wallet-derived keys

What is X1 Vault?

A decentralized encrypted storage system where your Solana/X1 wallet IS your encryption key. Files are encrypted client-side before upload — the server never sees your data.

  • ✓ End-to-end encryption (AES-256-GCM)
  • ✓ No accounts or passwords
  • ✓ Only your wallet can decrypt your files
  • ✓ Files stored on IPFS (content-addressed, permanent)

How It Works

┌─────────────────────────────────────────────────────────────────┐ │ ENCRYPTION FLOW │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 1. Sign Message 2. Derive Key 3. Encrypt │ │ ┌──────────────┐ ┌──────────────┐ ┌────────────┐ │ │ │ Wallet │───────▶│ SHA-256 │──────▶│ AES-256-GCM│ │ │ │ signs: │ │ (hash) │ │ │ │ │ │ "IPFS_ENC.." │ └──────────────┘ └────────────┘ │ │ └──────────────┘ │ │ │ │ │ │ ▼ │ │ │ │ ┌────────────┐ │ │ ▼ ▼ │ Encrypted │ │ │ Signature (64 bytes) AES Key (32 bytes) │ JSON │ │ │ └────────────┘ │ │ │ │ │ 4. Upload to IPFS ▼ │ │ ┌──────────────────────────────────────────────────────────┐ │ │ │ { version, algorithm, wallet, derivationMsg, data } │ │ │ └──────────────────────────────────────────────────────────┘ │ │ │ └─────────────────────────────────────────────────────────────────┘

Key Derivation

  1. Your wallet signs the message: IPFS_ENCRYPTION_KEY_V1
  2. The signature is hashed with SHA-256 → 32-byte AES key
  3. This key encrypts/decrypts all your files
  4. Same wallet = Same key = Access to all your files

File Format

{
  "version": 1,
  "algorithm": "AES-256-GCM",
  "wallet": "YourPublicKeyBase58...",
  "derivationMsg": "IPFS_ENCRYPTION_KEY_V1",
  "data": "base64(IV + ciphertext + authTag)"
}

The data field contains:

  • 12 bytes: Random IV (initialization vector)
  • N bytes: Encrypted content
  • 16 bytes: GCM authentication tag

API Reference

Endpoint Method Description
/api/v0/add?pin=true POST Upload file (multipart form). Headers: X-Pubkey, X-Filename, X-Content-MD5 (MD5 of plaintext before encryption)
/index/files?pubkey=X GET List all files for a wallet
/index/file/{cid} GET Get file metadata
/index/file/{cid} DELETE Remove from index (set X-Pubkey header)
/index/checksums?pubkey=X GET List MD5 checksums for all files (optional pubkey filter)

Upload Example (curl)

curl -X POST "https://vault.x1.xyz/ipfs/api/v0/add?pin=true" \
  -H "X-Pubkey: YourWalletPublicKey" \
  -H "X-Filename: myfile.txt" \
  -H "X-Content-MD5: $(md5sum plaintext.txt | cut -d' ' -f1)" \
  -F "file=@encrypted.json"

Note: The MD5 is of the unencrypted plaintext, sent via the X-Content-MD5 header at upload time. This allows duplicate detection even though each encryption produces different ciphertext (due to random IVs). Clients must compute and send this header — the server cannot derive it from the encrypted content.

Node.js

npm install @solana/web3.js tweetnacl bs58 form-data
#!/usr/bin/env node
const fs = require('fs');
const crypto = require('crypto');
const { Keypair } = require('@solana/web3.js');
const nacl = require('tweetnacl');
const bs58 = require('bs58');

const VAULT_URL = 'https://vault.x1.xyz/ipfs';
const DERIVATION_MSG = 'IPFS_ENCRYPTION_KEY_V1';

async function upload(keyPath, filename, data) {
    // Load key
    const keyFile = fs.readFileSync(keyPath, 'utf8');
    const match = keyFile.match(/SOLANA_PRIVATE_KEY=([^\s\n]+)/);
    const secretKey = bs58.decode(match ? match[1] : keyFile.trim());
    const keypair = Keypair.fromSecretKey(secretKey);
    const publicKey = keypair.publicKey.toBase58();

    // Derive AES key
    const sig = nacl.sign.detached(Buffer.from(DERIVATION_MSG), keypair.secretKey);
    const aesKey = crypto.createHash('sha256').update(sig).digest();

    // MD5 of plaintext (for duplicate detection)
    const contentMd5 = crypto.createHash('md5').update(data, 'utf8').digest('hex');

    // Encrypt
    const iv = crypto.randomBytes(12);
    const cipher = crypto.createCipheriv('aes-256-gcm', aesKey, iv);
    const encrypted = Buffer.concat([cipher.update(data, 'utf8'), cipher.final()]);
    const combined = Buffer.concat([iv, encrypted, cipher.getAuthTag()]);

    // Build payload
    const payload = JSON.stringify({
        version: 1, algorithm: 'AES-256-GCM', wallet: publicKey,
        derivationMsg: DERIVATION_MSG, data: combined.toString('base64')
    });

    // Upload
    const FormData = require('form-data');
    const form = new FormData();
    form.append('file', Buffer.from(payload), { filename: 'encrypted.json' });

    const resp = await fetch(`${VAULT_URL}/api/v0/add?pin=true`, {
        method: 'POST',
        headers: {
            'X-Pubkey': publicKey, 'X-Filename': filename,
            'X-Content-MD5': contentMd5, ...form.getHeaders()
        },
        body: form
    });
    return JSON.parse((await resp.text()).split('\n')[0]);
}

// Usage: upload('~/.env', 'file.txt', 'Hello world')

import os, json, base64, hashlib
from nacl.signing import SigningKey
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import base58, requests

VAULT_URL = 'https://vault.x1.xyz/ipfs'
DERIVATION_MSG = b'IPFS_ENCRYPTION_KEY_V1'

def upload(key_path, filename, data):
    # Load key
    with open(os.path.expanduser(key_path)) as f:
        content = f.read()
    import re
    match = re.search(r'SOLANA_PRIVATE_KEY=([^\s\n]+)', content)
    secret = base58.b58decode(match.group(1) if match else content.strip())
    
    signing_key = SigningKey(secret[:32])
    public_key = base58.b58encode(bytes(signing_key.verify_key)).decode()
    
    # Derive AES key
    signature = signing_key.sign(DERIVATION_MSG).signature
    aes_key = hashlib.sha256(signature).digest()
    
    # MD5 of plaintext (for duplicate detection)
    content_md5 = hashlib.md5(data.encode()).hexdigest()

    # Encrypt
    iv = os.urandom(12)
    aesgcm = AESGCM(aes_key)
    ciphertext = aesgcm.encrypt(iv, data.encode(), None)
    combined = base64.b64encode(iv + ciphertext).decode()

    # Build payload
    payload = json.dumps({
        'version': 1, 'algorithm': 'AES-256-GCM', 'wallet': public_key,
        'derivationMsg': 'IPFS_ENCRYPTION_KEY_V1', 'data': combined
    })

    # Upload
    resp = requests.post(
        f'{VAULT_URL}/api/v0/add?pin=true',
        headers={
            'X-Pubkey': public_key, 'X-Filename': filename,
            'X-Content-MD5': content_md5
        },
        files={'file': ('encrypted.json', payload)}
    )
    return resp.json()

# Usage: upload('~/.env', 'file.txt', 'Hello world')

class VaultClient {
    constructor(baseUrl = 'https://vault.x1.xyz/ipfs') {
        this.baseUrl = baseUrl;
        this.publicKey = null;
        this.derivedKey = null;
    }

    async connect() {
        const provider = window.x1_wallet || window.phantom?.solana || window.solana;
        if (!provider) throw new Error('No wallet');
        
        await provider.connect();
        this.publicKey = provider.publicKey.toString();
        
        const sig = await provider.signMessage(
            new TextEncoder().encode('IPFS_ENCRYPTION_KEY_V1'), 'utf8'
        );
        const sigBytes = sig instanceof Uint8Array ? sig : new Uint8Array(sig.signature);
        const hash = await crypto.subtle.digest('SHA-256', sigBytes);
        this.derivedKey = await crypto.subtle.importKey(
            'raw', hash, { name: 'AES-GCM', length: 256 }, false, ['encrypt', 'decrypt']
        );
    }

    async upload(content, filename) {
        const data = new TextEncoder().encode(content);

        // MD5 of plaintext (for duplicate detection)
        const contentMd5 = await this.md5(data);

        const iv = crypto.getRandomValues(new Uint8Array(12));
        const encrypted = await crypto.subtle.encrypt({ name: 'AES-GCM', iv }, this.derivedKey, data);
        const combined = new Uint8Array(iv.length + encrypted.byteLength);
        combined.set(iv);
        combined.set(new Uint8Array(encrypted), iv.length);

        const payload = JSON.stringify({
            version: 1, algorithm: 'AES-256-GCM', wallet: this.publicKey,
            derivationMsg: 'IPFS_ENCRYPTION_KEY_V1',
            data: btoa(String.fromCharCode(...combined))
        });

        const form = new FormData();
        form.append('file', new Blob([payload]));
        const resp = await fetch(`${this.baseUrl}/api/v0/add?pin=true`, {
            method: 'POST',
            headers: {
                'X-Pubkey': this.publicKey, 'X-Filename': filename,
                'X-Content-MD5': contentMd5
            },
            body: form
        });
        return JSON.parse((await resp.text()).split('\n')[0]);
    }

    async decrypt(cid) {
        const resp = await fetch(`https://ipfs.io/ipfs/${cid}`);
        const json = await resp.json();
        if (json.wallet !== this.publicKey) throw new Error('Wrong wallet');

        const combined = Uint8Array.from(atob(json.data), c => c.charCodeAt(0));
        const decrypted = await crypto.subtle.decrypt(
            { name: 'AES-GCM', iv: combined.slice(0, 12) },
            this.derivedKey, combined.slice(12)
        );
        return new TextDecoder().decode(decrypted);
    }

    // Compute MD5 hex digest in browser (no external deps)
    async md5(data) {
        // Uses a minimal pure-JS MD5 — or use a library like spark-md5
        // For production, consider importing a proper MD5 library
        const msgLen = data.length;
        const s = [7,12,17,22,7,12,17,22,7,12,17,22,7,12,17,22,
                   5,9,14,20,5,9,14,20,5,9,14,20,5,9,14,20,
                   4,11,16,23,4,11,16,23,4,11,16,23,4,11,16,23,
                   6,10,15,21,6,10,15,21,6,10,15,21,6,10,15,21];
        const K = Array.from({length:64},(_,i)=>Math.floor(2**32*Math.abs(Math.sin(i+1)))>>>0);
        const pad = new Uint8Array(msgLen+64-((msgLen+8)%64)+8);
        pad.set(data); pad[msgLen]=0x80;
        const dv = new DataView(pad.buffer);
        dv.setUint32(pad.length-8, (msgLen*8)>>>0, true);
        let [a0,b0,c0,d0] = [0x67452301,0xefcdab89,0x98badcfe,0x10325476];

For AI Agents & Bots

X1 Vault is designed for automated, non-interactive use. Your agent reads the private key from disk — no wallet popup, no user interaction.

Quick Integration

# Store your agent's wallet key
echo "SOLANA_PRIVATE_KEY=YourBase58Key..." > ~/.agent-wallet.env

# Upload encrypted file

# Or with Python

Why This Works for Agents

  • ✓ No browser required — pure CLI/script operation
  • ✓ Key from file — works in headless environments
  • ✓ Same encryption — compatible with web UI
  • ✓ Deterministic key derivation — same wallet = same encryption key every time

Multi-Agent Setup

Each agent gets its own wallet. Files are isolated by wallet — Agent A cannot decrypt Agent B's files.

# Agent A's files
python3 upload.py /keys/agent-a.env data.json "..."

# Agent B's files (separate wallet, separate encryption)
python3 upload.py /keys/agent-b.env data.json "..."

Backup Client (vault-backup)

Smart backup client that skips unchanged files using MD5 checksums. Computes MD5 of plaintext before encryption, checks the server index, and only uploads new or modified files.

How It Works

┌────────────────────────────────────────────────────────────────┐ │ BACKUP FLOW │ ├────────────────────────────────────────────────────────────────┤ │ │ │ For each local file: │ │ │ │ 1. Read file ──► MD5(plaintext) ──► "a1b2c3d4..." │ │ │ │ 2. GET /index/checksums?pubkey=... │ │ ┌──────────────────────────────┐ │ │ │ server.js md5:d38de5ec... │ ◄── already indexed │ │ │ config.json md5:a1b2c3d4... │ │ │ └──────────────────────────────┘ │ │ │ │ 3. Compare local MD5 against server index: │ │ • Match found ──► SKIP (no upload) │ │ • No match ──► Encrypt ──► Upload ──► Index │ │ │ │ Result: Only new/changed files are uploaded │ └────────────────────────────────────────────────────────────────┘

Dedup via X-Content-MD5

The backup client sends the X-Content-MD5 header with every upload. This is the MD5 of the plaintext before encryption. The server stores it in the index database.

On subsequent runs, the client fetches /index/checksums and compares each local file's MD5 against the server. Since the MD5 is of the plaintext (not the ciphertext), identical files always match — even though each encryption produces different ciphertext due to random IVs.

// Client-side dedup logic (simplified)
const content = fs.readFileSync('myfile.txt');
const md5 = crypto.createHash('md5').update(content).digest('hex');

const remote = await fetch('/index/checksums?pubkey=...');
const existing = remote.files.find(f =>
    f.filename.endsWith('myfile.txt') && f.md5 === md5
);

if (existing) {
    console.log('SKIP — unchanged');
} else {
    // encrypt, upload with X-Content-MD5 header
}

Integrating Into Your Own Client

Any client (Node.js, Python, Go, etc.) can implement the same dedup pattern:

  1. Compute MD5(plaintext) before encryption
  2. Check GET /index/checksums?pubkey=X for matching filename + MD5
  3. If match → skip. If no match → encrypt, upload with X-Content-MD5 header
# Python example
import hashlib, requests

def backup_file(server, pubkey, filepath, aes_key):
    with open(filepath, 'rb') as f:
        content = f.read()
    md5 = hashlib.md5(content).hexdigest()

    # Check server index
    checksums = requests.get(f'{server}/index/checksums?pubkey={pubkey}').json()
    filename = os.path.basename(filepath)
    for f in checksums['files']:
        clean = f['filename'].split('-', 1)[1] if '-' in f['filename'] else f['filename']
        if clean == filename and f['md5'] == md5:
            print(f'SKIP {filename} (unchanged)')
            return

    # Encrypt and upload (see Python example above)
    encrypted = encrypt(aes_key, content)
    resp = requests.post(
        f'{server}/api/v0/add?pin=true',
        headers={'X-Pubkey': pubkey, 'X-Filename': filename, 'X-Content-MD5': md5},
        files={'file': ('encrypted.json', encrypted)}
    )
    print(f'UP {filename} → {resp.json()["Hash"]}')

Security Model

What's Protected

  • ✓ File contents are encrypted before leaving your device
  • ✓ Only your wallet's signature can derive the decryption key
  • ✓ Each file has a unique random IV (no pattern analysis)
  • ✓ GCM mode provides authenticated encryption (tamper detection)

What's Visible

  • ⚠ Your wallet public key (who owns the file)
  • ⚠ File size (encrypted size ≈ original size)
  • ⚠ Upload timestamp (from index)
  • ⚠ Filename (if provided — consider omitting for sensitive files)

Trust Model

The server only stores ciphertext. It cannot:

  • Read your file contents
  • Derive your encryption key
  • Decrypt without your wallet

However, the server can delete files or refuse to serve them. For maximum resilience, pin your CIDs to multiple IPFS nodes.

Supported Wallets

  • X1 Wallet — Recommended for X1 chain
  • Phantom — Popular Solana wallet
  • Backpack — Multi-chain wallet
  • Any wallet supporting signMessage()

FAQ

Can I decrypt on a different device?

Yes! As long as you have access to the same wallet (same private key), you can decrypt your files from any device.

What if I lose my wallet?

If you lose access to your wallet's private key, your encrypted files are permanently inaccessible. There is no recovery mechanism — this is by design.

Can someone else decrypt my files if they know the CID?

No. They can download the encrypted JSON, but without your wallet's signature, they cannot derive the decryption key.

Is the filename encrypted?

No. Filenames are stored in plaintext in the index. For sensitive files, use generic names or leave the filename empty (the system will use the CID).
